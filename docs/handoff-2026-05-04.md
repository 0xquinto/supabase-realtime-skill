## Methodology notes (carry across sessions)

- **The spike-before-smoke sequence proved load-bearing.** Without the spike, the smoke would have shipped with single-INSERT@+100ms (silent flake) and count-only assertions (silent row-data strip). The recon's adversarial pass was right: variance-as-load-bearing was the actual ship-blocker, not central tendency.
- **Two-step substrate diagnosis.** Spike surfaced the symptom (`new: null`/`new: {}` on Edge). Probe (`eval/probe-edge-payload.ts`) decomposed root cause across 7 variants × auth/grant/RLS combinations. Probe is committed as evidence trail, not as a regression gate. This pattern is reusable for future substrate-diagnosis: spike measures, probe pins.
- **`gh api` verification on upstream PRs** — held this session (PR #2029 still OPEN as cited; PR #1580 still merged 2026-02-24). The discipline is sticky.
- **CLAUDE.md currency drift accumulates per ADR cycle.** Each ADR ship leaves the status block + repo-layout + ADR count behind. Worth a habit: every ADR-merge PR includes the relevant CLAUDE.md updates inline. Or every shipped ADR triggers a same-day mini-sweep PR. Both have precedent (`b5078a0`, `043cc75`, this session's #19).
