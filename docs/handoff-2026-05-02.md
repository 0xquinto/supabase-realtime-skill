## Conventions a new session must respect

All from prior handoff still hold. New ones surfaced this session:

- **Substrate-vs-composition split.** Substrate-correctness fixes ship with smoke-test receipts (ADR-0011, ADR-0013 pattern). Fixture-driven gates ship with the worked example. Don't roll the latter into the former — proxy gap, weak signal. ADR-0012 § 2 is the canonical writeup.
- **Branch-naming convention is `<prefix>/<topic>`** — `recon/`, `fix/`, `feat/`, `docs/` — *not* `adr-NNNN-<topic>`. The ADR number lives in the commit/PR title, not the branch name.
- **FAIL→fix→PASS smoke-test discipline** is mandatory for substrate-correctness ships. Write the smoke test against current code first, capture the FAIL receipt, land the fix, re-run, capture the PASS receipt. Both runs reproducible from the same branch's smoke test code (only the production source changes between runs).
- **ADR shape inventory** so a future session can pattern-match: accept (0001/0002/0003/0005/0009), partial-acceptance (0006/0007), reject (0008), proposed-deferral-with-rationale (0004/0012), predicted-and-confirmed-and-fixed (0011), predicted-and-empirically-refined (0013).
- **`httpSend()` failure-mode contract** is *not* what the `.d.ts` declares. The runtime rejects on non-202; the discriminated `success: false` branch is unreachable. Wrappers must be `try/catch`-shaped, not `if (!result.success)`-shaped.
- **`realtime.messages` RLS denial is silent.** REST returns 202, row is filtered out, no fan-out. Sender gets no error; receiver gets no message. Documented in `references/multi-tenant-rls.md` § "Failure mode: silent filtering, not loud rejection." If a future ADR needs an explicit ack, layer one on top — the substrate doesn't provide it.
- **`supabase-js` minimum is now `^2.88.0`** (Dec 2025). Below this, `httpSend` doesn't exist + the empty-Auth-header bug bites.
