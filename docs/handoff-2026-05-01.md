## Conventions a new session must respect

These are in `CLAUDE.md` already but worth re-stating since they've bitten in past sessions:

- `.ts` extensions on ALL relative imports (Deno bundler quirk).
- Vitest 2.x workspace via `vitest.workspace.ts` (not `test.projects` in `vitest.config.ts`).
- Realtime ~5s warm-up window — latency-sensitive evals must use a long-lived adapter and discard the first event.
- Smoke tests use `ResilientApiClient` + `fetchProjectKeys` + `serviceRole` key; production tool calls forward the agent's JWT.
- Don't put `process.env.X!` at module scope in eval scripts — read inside `main()` with explicit `process.exit(2)`.
- `vendor/foundation/` is a snapshot — don't edit; subclass test-locally or open an ADR.
- GH Actions `secrets.X` is forbidden in job-level `if:` — use a step that writes `secrets=true|false` to `$GITHUB_OUTPUT`.
- npm publishes via OIDC, not `NPM_TOKEN`.
- ADRs default to `Proposed`; only the operator marks them `Accepted`.
